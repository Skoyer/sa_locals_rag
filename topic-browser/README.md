# Topic Browser

Vite + React + TypeScript UI for browsing videos by **topic tags** (`topic_buckets`) and **clusters** (`cluster_id`). Data comes from `public/lessons.json`, generated by the Python export in [`web/summary_page.py`](../web/summary_page.py).

**`data/topic_tree.json` is not used for the topic sidebar** (that data is still only from `topic_buckets`). **Learning paths** (featured + user paths, watch progress) use **Supabase** — see [`LEARNING_PATHS_AUTH_SPEC.md`](./LEARNING_PATHS_AUTH_SPEC.md) for SQL and RLS.

## Generate data

From the repository root (with `playlist_archive.db` present):

```bash
python web/summary_page.py
```

This writes `web/data.js` and **`topic-browser/public/lessons.json`**.

## Supabase (sign-in, paths, watch progress)

1. Create a Supabase project and run the SQL + RLS from [`LEARNING_PATHS_AUTH_SPEC.md`](./LEARNING_PATHS_AUTH_SPEC.md).
2. In the **repository root** `.env`, set (Vite reads the parent directory — see `vite.config.ts` `envDir`):

```env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-publishable-anon-key
```

See [`.env.example`](./.env.example).

### Email sign-in only (magic link)

The app uses **email magic links** — no Google, GitHub, or passwords in the UI.

**In the Supabase dashboard:**

1. **Authentication → Providers → Email** — turn **Email** on (enabled by default on many projects).
2. **Authentication → URL Configuration**
   - **Site URL:** e.g. `http://localhost:5173` for local dev, or your production URL (e.g. `https://your-app.pages.dev`).
   - **Redirect URLs:** add the same URLs you use (localhost, `127.0.0.1`, and production). Magic links redirect here after the user clicks the email link.
3. **Email templates** (optional): under Authentication → Email Templates, you can customize the “Magic Link” email. The default works for development.

**How it works for users:**

1. Enter email → **Email me a link**.
2. Open the inbox → click **Log in** / **Confirm** in the Supabase email.
3. Browser opens your app URL with a session; the app loads and shows them signed in.

**Local testing:** If emails don’t arrive, check spam, or in Supabase **Authentication → Users** see whether the user was created. For faster iteration you can use [Inbucket](https://supabase.com/docs/guides/local-development) with local Supabase, or any real mailbox.

**Production (Cloudflare Pages, etc.):** Set **Site URL** and **Redirect URLs** to your real HTTPS origin. Restart the Vite build after changing `.env`.

You do **not** need Google Cloud or GitHub OAuth apps for this flow.

## Dev server

```bash
cd topic-browser
npm install
npm run dev
```

## Word cloud thumbnails

Lesson JSON references paths like `wordclouds/per_video/1.png`. For images to load during dev, either copy or symlink the repo `wordclouds/` folder into `topic-browser/public/wordclouds` (so files resolve at `/wordclouds/...`). Missing images are hidden on error.
